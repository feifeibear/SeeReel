ReelyAI Agent CLI

AI use me

This page is written for Codex, Claude Code, Cursor Agent, and other local AI agents. Use the CLI to turn a human video idea into a visible ReelyAI web workflow, then continue generation only after the human has configured Agent Plan access.

What you are allowed to do

Create the workflow

Create a session, write the natural-language idea as the logline, generate the script, and generate storyboard shots visible in the web UI.

Let the human take over

Return the session URL so the human can edit script, prompts, assets, shot durations, references, and generation choices.

Continue to video

After the human confirms credentials and prompt choices, render missing shots and stitch the final video through ReelyAI APIs.

Human checkpoint: Agent Plan

Do not invent, scrape, or expose API keys. The human must open and pay for Agent Plan, then enter the key themselves.
  1. Ask the human to open Volcengine Agent Plan and create an Agent Plan API key in the Ark console.
  2. Ask the human to open https://reelyai.app/, click the top-bar Agent Plan button, and paste the key for browser/manual generation.
  3. For CLI automation, the CLI also needs the key in its own local user scope. Ask the human to run the configure command themselves, or set REELYAI_AGENT_PLAN_TOKEN in the agent runtime environment.
reelyai configure --base-url https://reelyai.app --agent-plan-token "<human-pastes-key-here>"

The browser key and CLI key are separate because ReelyAI isolates visitors by cookie. Treat that isolation as a safety feature.

Install the CLI

Install from npm

npm install -g reelyai
reelyai --help

Install the AI skill bundled with the CLI

reelyai skill install --agent all
reelyai skill print

The npm package includes the reelyai-cli skill. Installing it gives Codex, Claude Code, Cursor Agent, and other local agent runtimes the operating procedure for using the CLI safely.

Repo-local fallback

git clone https://github.com/feifeibear/reelyai-agent.git
cd reelyai-agent
npm install
npm install -g ./packages/reelyai-cli
reelyai skill install --agent all

Create a reviewable workflow

Start here. This is low risk because it creates visible planning nodes and stops before paid video rendering.

reelyai workflow "一个失眠导演在午夜便利店遇见未来的自己" \
  --base-url https://reelyai.app \
  --duration 60 \
  --style "neo-noir, rain reflections, grounded sci-fi" \
  --json

Return the webUrl to the human. It should look like:

https://reelyai.app/#/s/ses_xxxxxxxx

Generate the full video

Only continue after the human confirms Agent Plan is configured and the web workflow looks acceptable.

reelyai render --session latest --stitch --json

Or create, render, and stitch in one run when the human explicitly asks for full automation:

reelyai workflow "用户的视频创意" --duration 60 --render --stitch --json

Poll until the CLI returns a final downloadUrl. If a shot fails, report the failed shot id and let the human edit or ask you to retry.

Fine-grained node operations

Use these when the human edits the workflow or asks you to operate one canvas node precisely.

Inspect one node

reelyai node get --id shot_xxxxxxxx --json
reelyai node get --id asset_xxxxxxxx --json
reelyai node get --id ses_xxxxxxxx --json

Update a video node prompt

reelyai node update-prompt --id shot_xxxxxxxx \
  --prompt "新的 Seedance 视频 prompt，保留角色服装和场景连续性" \
  --title "03 重新追上自己" \
  --duration 15 \
  --json

Generate or poll a single video node

reelyai node generate --id shot_xxxxxxxx --wait --json
reelyai node poll --id shot_xxxxxxxx --json

Generate a shot tailframe

reelyai node tailframe --id shot_xxxxxxxx --publish-tos --canvas-node --json

Use --publish-tos when the tailframe will become a Seedance reference. Remote Seedance cannot fetch local /media/... URLs.

Open VLM review and repair prompts

reelyai node review --id shot_xxxxxxxx --frame-count 8 --json
reelyai node repair --id shot_xxxxxxxx --json
reelyai final-review --session latest --json
reelyai final-review --session latest --repair --json

Assets and storyboards

reelyai node update-prompt --id asset_xxxxxxxx --prompt "角色参考图 prompt" --json
reelyai node generate --id asset_xxxxxxxx --model seedream-5-lite --json
reelyai node review --id asset_xxxxxxxx --json
reelyai publish-storyboards --session latest --json

Operational rules for AI agents

  • Use ReelyAI API/CLI state as the source of truth. Do not hide generated media in private scratch folders.
  • Refresh status before continuing after any human web edit.
  • Do not call paid video generation until Agent Plan is configured and the human has approved continuing.
  • For local development, replace the base URL with http://localhost:5173 or REELYAI_AGENT_BASE_URL.
  • If the public backend requires a shared access token, use REELYAI_ACCESS_TOKEN or --access-token.
ReelyAI keeps the web canvas human-editable: sessions, scripts, shots, assets, renders, stitch state, and final videos remain visible in the app.
