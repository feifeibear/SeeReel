# Vibe Creating Prompt Skill

## Goal

Vibe Creating 提纯用户真正想表达的内容，让视频模型更容易抓住画面中心、情绪方向和体验连续性。它强化故事、情绪、记忆、氛围、意象与主观体验；弱化低价值技术参数、机械拍摄执行语言和工业化分镜噪声。

## Quick Start

收到用户输入后按三步内部判断，不向用户暴露分类标签：

1. **场景是否适合 VC**：真实创意目标是否适合用故事、情绪、记忆、氛围、意象或体验流表达。
2. **表达该怎么处理**：当前输入是接近 VC、混合表达，还是精准控制表达；决定直接放行、轻度提纯、直接改写、补问、原样保留或给可选 VC 版。
3. **信息是否足够**：若缺少落地画面所需的关键信息，先问最少的 1–3 个问题，再改写。

## Routing Rules

### 高适配：VC 原生场景

适用于回忆、梦境、情绪短片、微叙事、主观体验、意象驱动、多镜头共同服务一个统一感受的内容。

- 接近 VC：直接改写；若原文已成熟，轻度提纯或直接放行。
- 混合表达：轻度提纯后改写，保留有效结构、叙事顺序和情绪推进。
- 精准控制写法：默认可 VC 转译；去掉低价值技术控制，保留情绪、叙事和观看感受。

### 部分适配：可选 VC 场景

适用于品牌展示、角色展示、风格化广告、产品氛围片、需要兼顾信息表达和情绪体验的内容。

- 接近 VC：轻度提纯；足够可用时直接放行。
- 混合表达：给可选 VC 版，让用户决定是否采用更强体验化表达。
- 精准控制写法：保留原意，提示如需要可额外提供 VC 转写。

### 低适配：不建议 VC

适用于功能演示、UI 教程、步骤说明、严格执行脚本、长篇精确对白同步内容。

- 默认原样保留或只做极有限清理。
- 说明该需求更适合传统分镜、教程脚本或执行稿工作流，不是否定创意本身。
- 只有用户明确要求时才局部风格化。

## Information Check

不要在关键信息缺失时强行改写。VC 提示词默认需要四层结构，缺哪层优先补哪层：

1. **视觉锚点**：最该被看见的核心，人、物、概念或特效本体。
2. **行为或状态**：正在发生什么，只写一个主动作或主状态。
3. **局部调性**：这一幕的感觉，一个氛围词或形容词即可。
4. **视频主题**：应用场景 + 画面风格，如概念短片、微叙事、影视预演、情绪表达；超写实、电影感、动画、东方写意、赛博、插画感等。

需要先补问的常见情况：只有抽象感受；没有明确主体；有主体但没有动作或状态；多镜头之间看不出关系；极短输入缺少风格方向或重点瞬间。通常只问一轮，除非缺口明显影响成片。

## Constraint Priority

用户硬约束优先级最高：

1. 明确指定内容与硬约束：台词、旁白、音乐、音效、歌词、口白、镜头结构、参数保留要求、交付格式、风格限制。
2. 创意优化：在不破坏约束的前提下提纯故事、情绪、记忆、意象和统一体验。
3. VC 范式一致性：最后才收束语言，让提示词更适合模型理解和生成。

规则：
- 台词、旁白、音乐、音效、歌词和口白应原样保留；可以重排位置，不能改写措辞、替换或删除。
- 如果画面适合 VC、声音不适合改写，只改画面部分。
- 用户明确要求保留编号、参数、结构或节奏点时，按要求保留；如另给 VC 版，作为额外版本。

## Camera Language Policy

不要一刀切删除镜头语言。保留或转译“让观众怎么感受”的镜头意图；默认降权或删除“告诉系统怎么拍”的低价值技术参数。

默认可删除或转译：焦段、毫米数、机位术语、运镜参数、镜头号、景深、光圈、曝光、快门、设备说明、A/B 机、coverage、纯剪辑指令。

若用户未声明是否保留精准控制：
- 默认不把技术控制当作硬约束。
- 可删除或转译成自然画面结果。
- 输出中简短说明已弱化、删除或转译技术控制；提示用户如需保留可继续指定。

## Rewrite Modes

按输入主导因素选择模式，不套固定模板：

- **叙事改写**：故事、关系、事件推进为主；保留顺序和情绪转折。
- **情绪改写**：氛围、感受、状态为主；强化环境、节奏、质感和观看感受，不硬补因果。
- **记忆改写**：回忆、闪回、旧时感、消逝感；保留模糊、缺失、脆弱和时间流失感。
- **意识流改写**：联想、碎片、主观感知；允许不完整，但画面仍要可感知，意象需统一。
- **多镜头体验改写**：多段、多场景、多切换共同服务同一体验；可自然分段，保留情绪递进和视觉母题。
- **混合提纯**：创意内容与执行语言混杂；移除技术噪声、重复说明和低价值控制，不擅自补新桥段。

## Output Rules

默认使用四段式输出，顺序固定：

1. **判断**：简要说明是否适合 VC、原文是否足够可用、信息是否充分。
2. **执行动作**：必须显式使用以下标签之一：直接放行 / 轻度提纯 / 直接改写 / 先补问 / 原样保留 / 可选 VC 版。
3. **输出结果**：给出改写结果、原样保留文本或补问内容。
4. **补充说明（如有）**：说明已转译的技术控制、已保留的声音/结构硬约束，或提示用户可继续指定保留项。

长度原则：
- 默认不要显著长于原文，不把极短输入扩成冗长散文。
- 不凭空增加人物关系、剧情反转、场景细节或情绪变化。
- 单段输出时尽量是一条可直接生成的提示词。
- 单幕信息充分且无额外约束时，通常控制在 30–120 字。
- 保留结构不等于保留编号；只有用户明确要求保留编号、列表或交付结构时才保留编号。

## Quick Reference

| 输入类型 | 默认动作 | 注意 |
|---|---|---|
| 主体、动作、氛围明确的单幕提示 | 直接改写 / 轻度提纯 / 直接放行 | 缺风格或画面中心时再问 |
| 多镜头共同服务统一体验 | 保结构改写 | 保留场景流转、情绪递进、视觉母题 |
| 分镜号和参数很多但底层是情绪故事 | 去噪后转译 | 删除参数时简短说明 |
| 品牌、角色、风格广告 | 轻度提纯 / 可选 VC 版 | 不强推体验化表达 |
| 只有“自由”“高级感”等抽象词 | 先补问 | 问视觉锚点、场景、动作或风格 |
| 含明确台词、旁白、音乐、音效 | 保留声音，只改画面 | 声音内容原样保留 |
| 用户明确要求保留参数或结构 | 原样保留 / 可选 VC 版 | 约束优先 |
| 功能演示、UI 教程、步骤说明 | 原样保留 | 建议走教程脚本，不做 VC |
| 长篇剧情且要求精确对白同步 | 不做 VC 改写 | 可建议拆纯画面段落 |
