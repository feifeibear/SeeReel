# 工作流式 UI 功能地图（实现状态与演进方向）

本文档用“工作流式 AI 视频画布”的通用能力清单，记录 cinema_agent / ReelyAI 的实现状态与后续补齐方向。它面向本项目自身定位做功能规划：以短剧生产管线为核心，把资产、分镜、视频生成、拼接与审片组织成可审计、可接管、可自动化的节点工作流。

- ✅ 已实现（位置 / 入口）
- 🟡 部分实现 / 已有相近替代（差距说明）
- ❌ 暂未实现（备注落地难点或建议路径）

> 方向：先稳住“画布 + 节点 + Inspector + API 可自动化”的主管线，再补通用媒体工具、模板、分组与时间轴能力。

---

## 1. 项目与画布

| 通用能力 | 状态 | 在本项目中的对应 |
|---|---|---|
| 一个画布 = 一个项目；项目可切换、删除、归档 | 🟡 | Session 列表代替项目库（左侧栏 `<aside class="sidebar">`）。Session 切换、删除、归档已具备 |
| 多类型节点承载文本、图像、视频、音频、脚本 | 🟡 | 当前节点按短剧生产语义划分：Asset（图/锚）/ Storyboard（图组）/ Shot（视频）/ Stitch（输出）/ Reference Video（参考视频）/ Video Processor（视频处理）。文本节点、音频节点仍可后续加入自由参考层 |
| 空白处右键创建节点 / 拖文件入画布 | ✅ | 右键菜单可建角色、场景、镜头、拼接节点，也可上传图片/视频；拖入图片/视频自动入库 |
| 节点复制、副本、删除、撤销 | 🟡 | Delete / Backspace 删除节点，顶部支持撤销/恢复；复制副本和保留连线还未完整产品化 |
| 小地图导航 + 缩放控制 | ✅ | xyflow `<MiniMap pannable zoomable>` + `<Controls>` |

## 2. 节点连线与工作流执行

| 通用能力 | 状态 | 对应 |
|---|---|---|
| 节点之间拉线 = 数据/参考依赖 | ✅ | Flow 视图中连线会写回实际状态：资产→分镜/视频、分镜板→视频、参考视频→视频、视频→拼接 |
| 用户手动布线 / 手动断线 | ✅ | `FlowView.onConnect` / `onEdgesDelete` 会写回 shot/session/stitch job 状态 |
| 多个输出节点 / 多条拼接路径 | ✅ | `Session.stitchJobs[]` 支持同一画布上多个独立拼接节点，每个节点有独立 playlist、输出、下载和终审 |
| 选中多节点打组 + 整体执行 | ❌ | xyflow 支持多选拖动，但没有“打组/子图模板”语义 |
| 保存可复用工作流模板 | ❌ | 暂无模板系统。建议在稳定节点语义后再做 |
| 重跑某个子图 | 🟡 | 单个节点 Inspector 可以重生/重拼；还没有“从某节点向下游重跑”的图级命令 |

## 3. 资产与图像工具

| 能力 | 状态 | 对应 |
|---|---|---|
| 文生图 / 图生图 / 多参考图 | 🟡 | Asset 支持文生、父资产参考和多图参考审计；融合/编辑能力仍可增强 |
| 角色/场景/道具/风格锚 | ✅ | Asset 类型承载跨镜一致性锚点，可通过 @mention 和连线复用 |
| 图像基础编辑：扩图、重绘、擦除、抠图、裁剪 | ❌ | 高频刚需，建议优先补在 Asset Inspector |
| 宫格/分镜板生成与引用 | ✅ | `sub-storyboard`、storyboard grid、shot-scoped sketches 已形成分镜参考链路 |
| 宫格切分 / 提取单格 | 🟡 | 内部已有 grid 生成与消费；面向用户的切分入口待补 |
| 风格库 / 一键预设 | ❌ | 可以先用 JSON 风格表 + 注入 composedPromptDraft 实现 |

## 4. 视频工具

| 能力 | 状态 | 对应 |
|---|---|---|
| 文生视频 / 首帧 / 首尾帧 / 参考视频 | ✅ | Shot 节点 + Seedance 后端支持多模式，服务器负责模式互斥 |
| 参考视频分析：拆镜头、提取时段、生成提示词草稿 | 🟡 | 上传视频后可分析成 `asset.parsedShots`，用于参考视频分析与提示词导入 |
| 视频片段裁取 + 出入点 | 🟡 | `extractTailVideoClip`、derive-clip/reclip 已有，时间轴式 UI 待补 |
| 多片拼接 + 自定义顺序 | ✅ | 多个 Stitch 节点支持独立 playlist 和输出 |
| BGM / 混音 / 字幕旁白 | 🟡 | narration 管线已有 TTS + 字幕视频；BGM/音效模型未接 |
| 视频高清、补帧、慢动作 | ❌ | 可通过 ffmpeg + 超分/补帧模型补齐 |
| 分离音视频、人声/背景声分离 | ❌ | 可作为 Video Processor 节点的后续工具 |

## 5. 可审计生成与人工接管

| 能力 | 状态 | 对应 |
|---|---|---|
| 生成前 prompt 预览 | ✅ | dryRun + composedPromptDraft，用户可编辑最终提交文本 |
| 历史版本 | ✅ | Shot renders 历史可查看、恢复、删除 |
| VLM 自审与重试 | ✅ | Asset/Shot/Final video 均有审片/自审字段和 UI |
| 网络/后台任务恢复 | 🟡 | Shot poll、Stitch/Narration fire-and-forget + poll；进程重启会标记 orphan task |
| 下载与播放代理 | ✅ | 同源 stream/download 路由规避远程 URL 过期/CORS 问题 |

## 6. 后续 ROI 排序

1. **图像基础编辑：抠图 / 重绘 / 扩图** —— 对出片质量和返工效率提升最大。
2. **风格库 + 一键预设** —— 小成本提升资产生成一致性。
3. **视频时间轴剪辑** —— 已有 ffmpeg 能力，补前端时间轴即可显著增强拼接前处理。
4. **打组 + 工作流模板** —— 把“角色 → 分镜板 → 视频 → 拼接”的子图保存为可复用模板。
5. **文本节点 + 音频节点** —— 进入自由参考层，不污染短剧主管线。
6. **多机位 / 角色三视图 / 光影矫正等快捷生成** —— 作为 Inspector 内快捷工具，不做竞品式命名。
7. **多模型切换器** —— schema 加 `modelId`，UI 加下拉。
8. **外部 3D/摄影预演工具接入** —— 作为上传参考图/视频，不在本项目内实现重型 3D 预演台。
